Core Modules

每一个模块都围绕“高并发下少拷贝、少锁、少系统调用、状态不撕裂”来设计。

⚙️

Single Reactor

主线程统一处理 epoll 事件，负责连接接入、读写事件重装、关闭 fd 和 timer 管理。

🧵

ThreadPool

工作线程消费 Task，解析 HTTP 请求，再通过 eventfd 通知主线程处理结果。

🧠

HTTP State Machine

手写请求行、请求头、Body 状态机，支持粘包续处理和连接状态复用。

🕒

TimerWheel

基于 timerfd 的 O(1) 时间轮，侵入式链表实现低开销连接超时剔除。

🛡️

Epoch Version

每个 Task 绑定连接版本号，防止 fd 复用后旧任务误操作新连接。

🚀

mmap + writev

文件映射进内存，响应头和文件内容聚合写出，让静态资源响应更干净利落。

HttpCode Connection::do_request() {
  // 1. build resource path
  stat(resource_path, &file_stat);

  // 2. open static file
  int fd = open(resource_path, O_RDONLY);

  // 3. map file into memory
  file_address = (char*)mmap(
    nullptr, file_stat.st_size,
    PROT_READ, MAP_PRIVATE, fd, 0
  );

  // 4. close fd after mmap
  close(fd);

  // 5. write header + file by writev
  make_response();
  return HttpCode::FILE_REQUEST;
}
